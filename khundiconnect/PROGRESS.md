### Permanent build-correctness rules (deploy.sh / build config) вЂ” `feat/25-build-reliability`
Two deploy-blocking bugs were found on staging and fixed permanently (alongside the existing **prisma-generate-before-build** and **npm-ci-include-dev** rules). Both are now enforced by `deploy.sh` and asserted before PM2 restarts, and documented in ARCHITECTURE.md В§28.5/В§28.6 + docs/AUTH.md:

1. **API incremental-no-op trap.** `nest build` with `deleteOutDir:true` + `"incremental": true` (in `apps/api/tsconfig.json`) + a leftover `*.tsbuildinfo` = an exit-0 build that wipes `dist/` then emits ZERO files, crashing the API with `Cannot find module './config/env.validation'`. **Fix:** removed `"incremental": true`; `deploy.sh` В§5a now purges `*.tsbuildinfo`, `apps/api/dist`, and the turbo cache (`.turbo`, `node_modules/.cache/turbo`) before building; `turbo.json` build `outputs` include `dist/**` (no more "no output files found" warning). `deploy.sh` В§5c asserts `apps/api/dist/main.js` AND `apps/api/dist/config/env.validation.js` exist, exiting non-zero otherwise so a broken build never restarts PM2.

2. **`NEXT_PUBLIC_*` not baked at build.** `next build` runs with `apps/web` as cwd and reads `.env` from the app dir (not the monorepo-root `.env`); turbo's strict env mode also strips undeclared vars. So a plain `turbo run build` did NOT inline `NEXT_PUBLIC_PLATFORM_BASE_DOMAIN`, silently killing the cross-subdomain login redirect (build-step 24) after deploy. **Fix:** `deploy.sh` В§5b writes an explicit ALLOWLIST (`NEXT_PUBLIC_PLATFORM_BASE_DOMAIN`, `NEXT_PUBLIC_OTP_LOGIN_ENABLED`, `NEXT_PUBLIC_MESSAGING_PROVIDER`) from the sourced `.env` into `apps/web/.env.local` before building; the same vars are declared in `turbo.json` build `env` for cache correctness. **Security:** the allowlist is the boundary вЂ” server secrets (`DATABASE_URL`, `JWT_*`, `ENCRYPTION_KEY`, `R2_*`) are NEVER copied. В§5c greps `.next/static` to confirm the public value baked in (fail loudly if absent) AND scans for leaked secret values (abort if any found). Verified: clean API build emits `main.js`+`config/env.validation.js`; a plain web build bakes the domain into `.next/static`; no server secret appears in `.next/static`.
