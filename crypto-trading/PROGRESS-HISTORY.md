### [2026-07-06] Module 02 — Database (Prisma + PostgreSQL) — DONE
**Built:** Full data model and money-critical ledger for the bot.
- `packages/db/prisma/schema.prisma` — 7 enums (`Mode`, `Tier`, `Strategy`, `PositionStatus`, `TradeResult`, `CycleStatus`, `Side`) and 10 models exactly per spec: `Setting`, `SettingAudit`, `Pair`, `Position`, `Fill`, `Trade`, `LevelingCycle`, `EquitySnapshot`, `EventLog`, `FilterState`. ALL monetary/quantity columns are `Decimal` (never Float). Every money row carries `mode`. Relations: `Fill → Position?`, `Fill → LevelingCycle?`, `Trade 1:1 Position` (`positionId @unique`). Added query indexes for the hot mode-scoped reads (`[mode,status]`, `[mode,symbol]`, `[mode,closedAt]`, `[category,ts]`, etc.).
- `packages/db/prisma/migrations/20260706000000_init/migration.sql` + `migration_lock.toml` (provider=postgresql). Generated deterministically via `prisma migrate diff --from-empty --to-schema-datamodel` (no live DB in the build env); 10 tables / 7 enum types / 16 indexes / 3 FKs. `prisma validate` ✅.
- `packages/db/src/index.ts` — process-wide `PrismaClient` singleton (`getPrisma`/`prisma`/`disconnectPrisma`), re-exports the generated enums + model types so downstream modules import DB types from `@spotedge/db` only.
- `packages/db/src/pnl.ts` — pure Decimal P&L math (the ledger's source of truth): `costBasis`, `proceeds`, `totalFees`, `grossPnl`, `realizedNetPnl` (= Σ sell qv − Σ buy qv − Σ feeUsd), `netPnlPct` (on cost basis), `classifyResult` (WIN/LOSS/BREAKEVEN/FORCED_REBUY_LOSS), `holdHours`, `computeTrade` (assembles the `Trade` row), `positionUnrealized` (spot long mark-to-market), `accountEquity` (allocated + realized + unrealized), and mode-isolation helpers `scopeToMode`/`realizedNetPnlForMode`.
- `packages/db/src/settings.ts` — typed `Setting` access; `setSetting` upserts + journals `SettingAudit` (old→new, source) in ONE transaction; `DEFAULT_SETTINGS` (mode, capital_paper/live, risk_pct, summary_time_pkt, strategy_A/B/C_enabled, circuit_breaker_tripped_date, live_gate_approved, bnb_fee_reserve_usd); `parseBool`/`parseNum` helpers.
- `packages/db/src/events.ts` — `makeEventSink(prisma)` returns the function the bot's logger `attachEventSink()` seam consumes to persist `EventLog` rows (never throws into the engine; folds `type` into `data` JSON). Defined `EventRecordLike` locally to avoid a bot→db import cycle.
- `packages/db/prisma/seed.ts` — idempotent seed: settings create-only (never clobbers an operator-edited value), 15 default pairs upserted by symbol with tiers (T1 BTC/ETH/BNB/SOL · T2 DOT/LINK/AVAX/NEAR/TON/SUI/FIL/ATOM · T3 XLM/DASH/ALGO; GRAM+CVX excluded). Wired via `package.json#prisma.seed = tsx prisma/seed.ts`.
- `scripts/backup-db.sh` — nightly `pg_dump | gzip -9` to `${BACKUP_DIR:-~/spotedge-backups}/spotedge-YYYYMMDD.sql.gz` (dir OUTSIDE repo), `--no-owner --no-privileges` for scratch-DB restore, `.partial`→atomic rename, 7-day retention prune, non-zero exit on failure.
- `apps/bot/src/backup.ts` (`BackupJob`) — in-bot scheduler fires the script at **03:30 UTC** (reuses scheduler `nextPeriodic`/`DAY_MS`; injectable clock/runner for tests); `runOnce()` never throws — logs `backup_ok`/`backup_failed` (Telegram alert deferred to module 11). Wired into `apps/bot/src/index.ts` boot (step 5b) + graceful-shutdown hooks (`backup.stop()`).
- `deploy.sh` — activated the module-01 Prisma placeholders: `prisma generate` → `migrate deploy` → `seed` (via `pnpm --filter @spotedge/db exec/run`), AFTER install and BEFORE typecheck/build, per AGENT.md rule (b). CI == the SSH deploy workflows that invoke `deploy.sh`, so they inherit the same steps (no separate build/test CI workflow exists in this repo).
- `packages/db/package.json` — added `@prisma/client`+`decimal.js` deps, `prisma`+`tsx`+`vitest` dev-deps, and `generate`/`migrate:*`/`seed`/`test` scripts; `build`/`typecheck` now run `prisma generate` first so the client exists before tsc.
