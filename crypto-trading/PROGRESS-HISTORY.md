### [2026-07-06] Module 03 — Exchange Connector (Binance Spot) — DONE
**Built:** The single code path that talks to Binance — `BinanceConnector` (ccxt REST + `ws` streams) living in `@spotedge/core`, environment-agnostic and fully seam-injected so `packages/core` never imports `apps/bot`. Everything above this layer is exchange-agnostic and depends only on `exchange/types.ts`.
- `packages/core/src/exchange/types.ts` — the exchange contract: `Timeframe`+`TF_MS`, `Candle`, `Ticker`, `ExchangeFilters` (all decimal STRINGS), `FillReport`/`FillLeg`, `Balance`, `OpenOrder`, `OrderPurpose`, `ReconcileAction`/`ReconcileFinding`, and the injected seams `ExchangeLogger`, `AlertFn`, `Clock`(+`systemClock`), `SettingsReader`, `PairRepository`, `OpenPositionView`. Market-data OHLCV are `number` (indicator math, not ledger money); everything that touches the ledger is a decimal string.
- `packages/core/src/exchange/filters.ts` — PURE exchange-filter compliance (the classic bot killer): `roundQty`/`roundPrice` (floor to stepSize/tickSize at exact step precision, decimal-safe), `meetsMinNotional` (minQty AND qty×price≥minNotional), `validateTp1Split` (BOTH halves of a 50% TP1 split must clear minNotional at entry — the veto the risk engine/module 06 calls), `stopLimitPrice` (limit = trigger×0.995 buffer), `isTradable`.
- `packages/core/src/exchange/clientOrderId.ts` — idempotency keys `se-{refId}-{purpose}-{nonce}`; `makeClientOrderId` (throws rather than truncate past Binance's 36-char cap), `parseClientOrderId`/`isOwnClientOrderId` (foreign/manual orders → null → reconcile ignores), `nextNonce` (ms+counter base36).
- `packages/core/src/exchange/backoff.ts` — WS reconnect schedule: `backoffDelay` (deterministic 1s→60s cap, overflow-safe) + `Backoff` class (next/reset, optional injectable full-jitter).
- `packages/core/src/exchange/fees.ts` — fee→USD so every `Fill.feeUsd` is accurate: `feeToUsd` (stablecoins 1:1, else `UsdPriceResolver`; unknown price → undefined so the caller alerts rather than under-count), `aggregateFeeUsd` (per-asset totals + summed USD + `unpriced` list), `STABLE_USD_ASSETS`.
- `packages/core/src/exchange/reconcile.ts` — pure `classifyReconcile(qtyRemaining, freeBalance, stopAlive, dustEps)` → `EXTERNAL_CLOSE` (balance gone) | `REPLACE_STOP` (stop missing) | `OK`, with a minQty dust epsilon to avoid false external-closes.
- `packages/core/src/exchange/ExecutionRouter.ts` — the PAPER/LIVE order seam: `ExecutionRouter` interface + `OrderContext`, and `UnwiredPaperRouter` that FAILS CLOSED (throws "wire module 10") so nothing silently trades into a void before the paper engine exists.
- `packages/core/src/exchange/BinanceConnector.ts` — the class: `init()` (loadMarkets → cache filters → clock check → arm hourly-drift + daily-refresh timers, all `unref`'d), `refreshFilters()` (persists `Pair.exchangeFilters`, detects status≠TRADING → mark `removing` + alert, skips testnet-absent pairs), `getKlines()` (CLOSED candles only — drops the forming bar by closeTime>now), `get24hVolumeUsd()`, `getBalances()`, order methods routed through the ExecutionRouter (`marketBuy`/`marketSell`/`placeStopLimit`/`cancelOrder`/`getOpenOrders`), a private `LiveExecutionRouter`+`live*` executors (idempotency query-before-place via `findByClientOrderId`; market buy by quote cost via `createMarketBuyOrderWithCost`; STOP_LOSS_LIMIT with 0.5% buffer; `settleOrder` re-queries + alerts on non-FILLED, spec retry-once), `toFillReport` (weighted-avg price + USD fees from real `info.fills`), WS ticker stream (miniTicker combined stream, exponential-backoff auto-reconnect, resubscribe, REST 10s poll fallback while down, >60s data-gap alert), user-data stream (listenKey + 30-min keepalive), `checkClockDrift()` (>1s → alert with chrony/timesyncd hint, recvWindow 10000), `checkBnbReserve()` (alert-only below `settings.bnb_fee_reserve_usd`; never auto-buys BNB), `reconcile(positions, handler)` (LIVE-only; delegates ledger effects to an injected `ReconcileHandler`, downgrades failed stop re-place to WATCHDOG_ONLY), plus the exposed rounding helpers + `validateSymbol()` (ADDPAIR). Constructed cheaply (no network); `init()` does first I/O. Test seam: inject a pre-built ccxt exchange.
- `packages/core/src/utils/money.ts` — decimal-safe primitives for core (core can't import apps/bot's money util): `dec/add/sub/mul/div/gte/lte/gt/lt`, `roundDownToStep`, `stepDecimals`, `toFixed/toStr/toNumber`. This is the module that introduces the exchange-precision rounding AGENT.md rule 7 refers to.
- `packages/core/src/index.ts` — re-exports the whole `exchange/*` surface + `money` namespace for downstream modules.
- `apps/bot/src/exchange.ts` — production wiring: adapts pino (`categoryLogger("exchange")`) to `ExchangeLogger`, a log-only `AlertFn` that routes through `logEvent` (→ events_log; module 11 fans out to Telegram), a Prisma-backed `PairRepository` (`listPairs`/`saveFilters`/`markRemoving`), a DB `SettingsReader` (`getSetting`+`parseNum`/`parseBool`), and `createExchangeConnector(cfg)` which CONSTRUCTS (network-free) but does not `init()`.
- `apps/bot/src/index.ts` — boot step 3 now constructs the connector (replacing the module-01 clock-drift stub) and logs "init deferred to trading start"; `exchange.close()` added to the graceful-shutdown hooks (tears down WS + timers, leaves exchange stop orders in place).

- `packages/db/src/settings.ts` — typed `Setting` access; `setSetting` upserts + journals `SettingAudit` (old→new, source) in ONE transaction; `DEFAULT_SETTINGS` (mode, capital_paper/live, risk_pct, summary_time_pkt, strategy_A/B/C_enabled, circuit_breaker_tripped_date, live_gate_approved, bnb_fee_reserve_usd); `parseBool`/`parseNum` helpers.
- `packages/db/src/events.ts` — `makeEventSink(prisma)` returns the function the bot's logger `attachEventSink()` seam consumes to persist `EventLog` rows (never throws into the engine; folds `type` into `data` JSON). Defined `EventRecordLike` locally to avoid a bot→db import cycle.
